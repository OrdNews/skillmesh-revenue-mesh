# Judge Pitch

## 15-Second Version

SkillMesh Revenue Mesh is an onchain operating budget system for self-funding agent organizations on X Layer. It scouts treasury context, routes capital, spends its own budget to hire specialist agents over x402, and leaves a proof-backed operating record onchain.

## 30-Second Version

Most agent demos show isolated capability: a task bot, a trading bot, or a payment demo.

SkillMesh Revenue Mesh shows organizational capability.

It gives an onchain organization:

- treasury
- role separation
- machine budget allocation
- specialist hiring
- proof-backed receipts

So the question is no longer, `can an agent act?`
The question becomes, `can an agent organization manage its own budget and prove its operations?`

## 60-Second Version

SkillMesh Revenue Mesh is our X Layer Arena submission for the OKX Build X Hackathon.

We built a self-funding agent organization on X Layer:

- `Scout` reads treasury and market context
- `Trader` prepares a live route
- the organization spends treasury budget over `x402` to hire specialist agents
- `Treasury` records task proof and mints a receipt onchain

This is not a generic task marketplace and not a pure trading bot.

What is new here is that spending is tied back to an operating record:

- who spent
- why they spent
- which specialist was hired
- which proof was written

That makes SkillMesh Revenue Mesh feel less like an agent feature and more like a real operating system primitive for autonomous organizations.

## The question to plant in the judge's mind

`What happens when organizations, not just agents, become autonomous onchain actors?`

## The sentence to end on

`We are not showing that agents can pay. We are showing that agent organizations can manage budget, buy intelligence, and prove their operations.`
